# Code Review Checklist

MR 코드 리뷰 시 확인해야 할 핵심 항목들입니다.

## 1. 아키텍처 & 설계

- [ ] 변경사항이 프로젝트 아키텍처를 따르는가?
- [ ] 레이어 분리가 올바른가? (Controller/Service/Repository)
- [ ] 의존성 방향이 올바른가?
- [ ] 단일 책임 원칙을 지키는가?
- [ ] 불필요한 복잡도가 없는가?

## 2. 코드 품질

- [ ] 변수/함수/클래스명이 명확한가?
- [ ] 매직 넘버/문자열이 상수로 정의되었는가?
- [ ] 중복 코드가 없는가?
- [ ] 함수가 너무 길지 않은가? (30줄 이하 권장)
- [ ] 주석이 필요한 복잡한 로직에 설명이 있는가?

## 3. 에러 처리

- [ ] 예외 상황이 적절히 처리되는가?
- [ ] 에러 메시지가 명확한가?
- [ ] Try-catch가 적절히 사용되는가?
- [ ] 에러가 적절히 로깅되는가?
- [ ] 사용자에게 친화적인 에러 메시지를 제공하는가?

## 4. 보안

- [ ] SQL Injection 위험이 없는가?
- [ ] XSS 취약점이 없는가?
- [ ] 민감 정보(API 키, 비밀번호)가 하드코딩되지 않았는가?
- [ ] 인증/인가가 적절히 구현되었는가?
- [ ] 입력 검증이 충분한가?

## 5. 성능

- [ ] N+1 쿼리 문제가 없는가?
- [ ] 불필요한 데이터베이스 호출이 없는가?
- [ ] 메모리 누수 가능성이 없는가?
- [ ] 비효율적인 루프가 없는가?
- [ ] 캐싱이 적절히 사용되는가?

## 6. 테스트

- [ ] 새로운 기능에 테스트가 추가되었는가?
- [ ] 테스트가 성공/실패/Edge case를 커버하는가?
- [ ] 기존 테스트가 여전히 통과하는가?
- [ ] 테스트 커버리지가 충분한가?
- [ ] Mock이 적절히 사용되었는가?

## 7. 가독성 & 유지보수성

- [ ] 코드가 읽기 쉬운가?
- [ ] 일관된 코딩 스타일을 따르는가?
- [ ] Import 순서가 정리되어 있는가?
- [ ] 사용하지 않는 코드가 제거되었는가?
- [ ] TODO/FIXME가 적절히 관리되는가?

## 8. 비즈니스 로직

- [ ] 요구사항을 충족하는가?
- [ ] Edge case가 처리되는가?
- [ ] 비즈니스 규칙이 올바르게 구현되었는가?
- [ ] 데이터 일관성이 유지되는가?
- [ ] 트랜잭션 처리가 적절한가?

## 위험도 판단 기준

### 🔴 Critical (반드시 수정)
- 보안 취약점
- 데이터 손실 위험
- 프로덕션 장애 가능성
- 심각한 성능 문제

### 🟡 High Priority (수정 강력 권장)
- 버그 가능성
- 코드 품질 저하
- 유지보수 어려움
- 테스트 누락

### 🟢 Medium Priority (개선 권장)
- 네이밍 개선
- 코드 중복 제거
- 주석 추가
- 리팩토링 제안
